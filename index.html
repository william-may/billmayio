Projects

Manage Registrations

  • Pending Registrations

    Any person who wishes to utilize the system must register to use it. Authorized system users are alerted there are registrations pending. The authorized user accesses the Administration module/Registration functionality and can Approve or Deny the registration.

  • Approve Registration

    If the authorized system user has approved the new user’s registration, they are brought to the Edit User page. This page displays a list of Roles which can be assigned to the new user. This system is role/permission based, that is, the Roles grant system permissions. Once the authorized system user has approved the new user’s registration and assigned roles applicable to support their business usage of the tool, they submit the content. Certain Roles dictate that a VISN and/or Facility be populated for the user, others do not.

  • Deny Registration

    If the authorized system user has denied the new user’s registration, they are brought to the Edit User page. This page now displays a field entitled ‘Registration Denied Reason’ which the authorized system user must select a value for. Typically no role is assigned and often, the new user’s profile will be Inactivated or Locked.

Quality Improvement

The Quality Improvement Reports module enables the VA Home Telehealth program personnel to create and track issue with Vendor equipment. The QIRs have a lifecycle; New, Approved, Replied, Agreed and Closed. They can also be Withdrawn.

  • Create New QIR

    This page must be populated by VA personnel who are identifying an issue with Home Telehealth Vendor equipment in the field. All mandatory fields are marked with red asterisk.

  • Search For QIR

    This page allows VA personnel to search the entire database of all QIR records for all Vendors. They can narrow search results by QIR number, Date range, Status, Vendor, Facility, etc. Results are displayed in the table.

  • Edit QIR

    To move a QIR from one point in the lifecycle to the next, the authorized user will open the EDIT QIR page. Here they can enter additional textual information to be shared with the Vendor, they can attach documents as needed and they can advance the QIR to the next point in the lifecycle.
